# Final QA Checklist

Before returning a final HTML deck, check the following:

## Structure
- Each slide or section is a 16:9 page or behaves as one full screen.
- Directory links jump to the correct page or chapter.
- Progress indicator updates correctly.
- Keyboard navigation works if included.

## Visual fit
- No page is clipped at the top or bottom.
- No important text is hidden by navigation, sidebar, or progress elements.
- Sparse pages do not feel empty.
- Dense pages are not forced into tiny text.

## Typography
- Titles are consistent across page types.
- Body text is readable on a projected screen.
- Tables and code blocks wrap properly.
- Avoid blur from large transforms or filters.

## Interaction
- Hover and click effects are subtle.
- Animations do not distract from the course content.
- Do not add unnecessary controls.

## Delivery
- If returning a file, make it a single self-contained HTML file when possible.
- If returning code, include the full document from doctype to closing html tag.
