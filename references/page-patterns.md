# Page Patterns

## Cover page

Use for course title and subtitle.

Structure:

- top small label
- large gradient title
- subtitle
- right-side soft visual or central card
- minimal footer if needed

## Directory page

Use when showing PART list only.

Structure:

- title: 目录
- one large catalog card
- rows for each PART
- number badge + PART label + title

Do not add extra explanation when user says only show directory.

## Chapter divider page

Use when transitioning between PARTs.

Structure:

- central card
- chapter label
- large number badge
- PART label
- chapter title

Keep it sparse and centered.

## Content matrix page

Use for “人群 × 用法”, “角色 × 场景”, “能力 × 场景”.

Structure:

- header title and one-sentence lead
- main card
- optional matrix header row
- 2x2 or 3x2 cards
- bottom summary sentence if helpful

## Comparison page

Use for “差 vs 好”, “不会用 vs 会用”, “AI 先做 vs 人把关”.

Structure:

- left negative card in red/amber soft tone
- right positive card in teal/blue soft tone
- optional center arrow or key shift card
- concise bottom insight

## Process / flow page

Use for steps, workflow, evolution, plan mode.

Structure:

- 3 to 6 step cards
- numbered step badges
- arrows or simple connectors
- bottom summary

## Table page

Use for file explanations, checklists, comparisons with many text items.

Structure:

- single large table card
- high contrast header row
- icon/file badge in first column
- explanation in second column

Prefer one large table over many small cards when the user asks for a table.

## Image showcase page

Use when user provides screenshots to embed.

Structure:

- title optional
- one or two large image containers
- `object-contain`, no cropping
- if privacy is requested, mask or replace sensitive data before embedding

## Ending page

Use only one central sentence when requested.

Structure:

- central card
- optional rocket/check icon
- large gradient sentence
- optional small decorative gradient line
